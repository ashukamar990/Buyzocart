## 2025-05-15 - DOM-based XSS in SPA Rendering Sinks
**Vulnerability:** Multiple Cross-Site Scripting (XSS) vulnerabilities were found in `main.js` and `admin.html` where dynamic data (notifications, search suggestions, and dashboard tables) was injected into `.innerHTML` without sanitization.
**Learning:** Vanilla JavaScript SPAs often use template literals with `innerHTML` for rapid UI rendering. Without a consistent sanitization layer, any data sourced from users or a shared database (like Firebase) becomes a vector for executing malicious scripts across different user sessions.
**Prevention:** Standardize a centralized `escapeHTML` utility and mandate its use for all dynamic data bound to `innerHTML` sinks. Whenever possible, prefer `.textContent` or `.innerText` for plain text to eliminate the risk by design.
